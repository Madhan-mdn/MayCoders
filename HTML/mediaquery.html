What is Media Query?

Media queries are the backbone of Responsive Web Design (RWD), allowing web pages to adapt their layout and styling in real-time based on the user's device characteristics, viewport size, or system preferences.

Declaration

They use the @media CSS at-rule to apply a specific block of CSS rules only if the device meets the defined condition.

Syntax Structure

A media query consists of an optional media type (e.g., screen for displays, print for documents) and a media feature condition enclosed in parentheses.

Breakpoints

The width or height condition where a web layout fundamentally changes to fit the screen is called a breakpoint. Common breakpoints include widths for mobiles (~ 480px), tablets (~ 768px), and desktops (~ 1024px).

Logical Operators

Conditions can be chained using operators like and (both conditions must be true), not (reverses the query), or only (ensures legacy browsers ignore the styles).

Beyond Screen Width

While screen width and height are the most common uses, media queries also detect device orientation (portrait/landscape), resolution (for Retina displays), and system-level user preferences (like prefers-color-scheme for dark mode).

CSS Cascade & Specificity

Media queries do not add specificity to the selectors inside them. Because source order matters in CSS, any layout overrides must be placed correctly (usually sequentially by size) so the browser cascades them accurately.
